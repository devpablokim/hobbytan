OpenClaw + Discord로 할 수 있는 일

실무 기준으로 보면 핵심은 4가지입니다. 채팅으로 지시하고, 여러 역할의 에이전트가 나눠서 일하고, 필요한 곳은 승인받고, 결과는 다시 Discord로 보고합니다.

1. 채팅으로 실행

"랜딩페이지 만들어줘", "문의 폼 붙여줘", "이 보고서 OCR해서 웹에 올려줘" 같은 자연어 지시로 작업을 시작합니다.

2. 역할 분리 협업

DEO는 조율, PM은 구조, PO는 검수, DEV는 구현과 배포를 담당하는 식으로 멀티 에이전트 협업이 가능합니다.

3. 실제 도구 연동

GitHub, Firebase Hosting, Firestore, 브라우저, 파일 시스템, 메시징을 연결해 결과물이 실제 서비스에 반영됩니다.

4. 승인 기반 안전장치

민감하거나 외부 영향이 있는 작업은 승인과 검토를 거치며, 연결된 권한 범위 안에서만 동작합니다.

처음 시작하는 방법

이 페이지는 도입 시작 페이지입니다. 실제 세팅은 아래 순서대로 진행하면 됩니다.

1

OpenClaw 실행 환경 준비

OpenClaw를 설치하고 실행 가능한 서버 또는 로컬 환경을 준비합니다. 문서와 예제 설정 파일을 기준으로 기본 런타임을 먼저 확인합니다.

2

Discord 연결

Discord 봇 또는 연결 채널을 구성하고, 작업을 받을 서버/채널을 지정합니다. 운영 채널, 보고 채널, 테스트 채널을 분리하는 것이 좋습니다.

3

권한과 키 연결

GitHub push, Firebase Hosting 배포, Firestore rules, 메시징 권한 등 실제 작업에 필요한 연결을 준비합니다. 현재 hobbytan 환경은 `main` 브랜치 + `firebase-static.json` 기준 배포로 운영 중입니다.

4

역할 분리

DEO(조율), PM(기획), PO(검수), DEV(구현·배포)처럼 역할을 나누면 복잡한 작업도 병렬로 진행하기 쉬워집니다.

5

첫 작업 실행

작게 시작합니다. 예를 들어 랜딩페이지 1개, 문서 OCR 1건, 문의 폼 1개처럼 즉시 검증 가능한 태스크로 시작하는 것이 좋습니다.

실전 명령 예시

@DEV-TAN /openclaw/howtostart 페이지 만들어줘 @DEO-TAN 이 보고서 OCR해서 JSON/MD로 만들고 웹에 배포해줘 @PM-TAN 이 페이지 구조를 기획적으로 보강해줘 @PO-TAN 라이브 브라우저로 풀페이지 검수해줘

운영 원칙

보고는 간결하게

작업 중계보다 완료 보고 중심으로 운영합니다. 채널을 로그로 채우지 않고, 결정과 산출물 중심으로 커뮤니케이션합니다.

민감 작업은 승인 기반

배포, 삭제, 외부 전송 등은 검토와 승인을 거쳐야 합니다. AI는 자동화되지만 무제한 권한으로 동작하지 않습니다.

최종 검토는 사람 책임

AI가 초안을 빠르게 만들 수는 있어도, 중요한 문구·수치·대외 발표물은 사람이 최종 확인해야 합니다.

연결된 권한 안에서만 실행

OpenClaw는 연결된 repo, 배포 계정, 메시지 채널 등 실제 허용된 범위 안에서만 일합니다.

현재 연결 상태 예시

hobbytan 웹 환경

Repo: devpablokim/hobbytan
Branch: main
정적 배포: firebase-static.json
Firebase 프로젝트: hobbytan-ai

DEV-TAN 보유 상태

Git push, Firebase Hosting 배포, Firestore rules 배포, `.env.local`, `functions/.env` 접근 경로 확인 완료 상태입니다.

FAQ

Q. OpenClaw는 모든 걸 자동으로 해주나요?

아닙니다. 반복 작업과 초안 생성, 연결된 도구 실행을 빠르게 도와주지만, 중요한 의사결정과 최종 검토는 사람이 해야 합니다.

Q. Discord만 있으면 바로 되나요?

Discord는 인터페이스일 뿐입니다. 실제로는 OpenClaw 실행 환경, 키, 권한, 배포 경로가 함께 준비되어야 합니다.

Q. 어디까지 연결 가능한가요?

GitHub, Firebase, 웹 브라우저, 파일 시스템, 메시징 등 연결된 범위 안에서 자동화가 가능합니다.

Q. 도입은 어떻게 시작하나요?

가장 좋은 시작은 작은 업무 하나를 정해 실제로 자동화해보는 것입니다. 예를 들면 랜딩페이지 제작, 문의 자동화, OCR 뷰어 제작처럼 바로 검증 가능한 작업이 좋습니다.
